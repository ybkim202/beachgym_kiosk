# 일산비치짐 키오스크 — 개발자 인수인계

해파랑 웰니스파크(일산해수욕장 · The Deck) 무인 키오스크 + 관리자 + 설문 + 직원 업무기록 웹앱입니다.
이 문서 하나로 **호스팅 + DB 연결**까지 끝낼 수 있게 정리했습니다.

---

## 1. 기술 스택 / 실행

- **Next.js 16 (App Router) · React 19 · TypeScript · Tailwind v4**
- **Node 20+ 필요** (개발은 Node 24에서 진행)
- 패키지 매니저: npm

```bash
npm install
npm run dev      # 개발 (localhost:3000)
npm run build    # 프로덕션 빌드
npm run start    # 프로덕션 실행
```

빌드/타입 모두 통과한 상태입니다 (`npm run build` 성공).

## 3. ⚙️ 환경변수 (.env.local 또는 호스팅 환경변수)

`.env.example` 참고. **필수 4개**만 넣으면 동작합니다.

```bash
# ---- 필수 ----
SUPABASE_URL=https://xxxx.supabase.co
SUPABASE_SERVICE_ROLE_KEY=...        # Supabase Settings > API > service_role (또는 secret 키). 서버 전용, 노출 금지
ADMIN_PASSWORD=관리자_비밀번호          # /admin, /staff 공용
ADMIN_SECRET=긴_랜덤_문자열            # 세션 쿠키 서명용

# ---- 카카오 알림톡 (선택, 미설정 시 발송 생략하고 체크인은 정상) ----
NOTIFY_PROVIDER=solapi               # solapi | sms | console
SOLAPI_API_KEY=
SOLAPI_API_SECRET=
SOLAPI_SENDER=01012345678            # 사전 등록 발신번호
KAKAO_PF_ID=                         # 카카오 비즈채널 발신프로필 키
KAKAO_TEMPLATE_ID=                   # 승인된 알림톡 템플릿 ID

# ---- 외부 링크 (선택, QR·알림톡에 사용) ----
NEXT_PUBLIC_HOMEPAGE_URL=https://www.ilsanbeachgym.com
NEXT_PUBLIC_RESERVATION_URL=https://www.ilsanbeachgym.com/   # 클래스 예약(완료화면 QR)
NEXT_PUBLIC_SURVEY_URL=https://<배포주소>/survey              # 카톡으로 보낼 설문 링크 → 반드시 배포 도메인으로
NEXT_PUBLIC_GUIDE_URL=https://www.ilsanbeachgym.com/guide
```

> ⚠️ `SUPABASE_SERVICE_ROLE_KEY` 는 **서버 전용 비밀키**입니다. `NEXT_PUBLIC_` 안 붙음 → 브라우저에 노출 안 됨. 절대 깃/외부에 올리지 마세요. (`.env.local` 은 `.gitignore` 처리됨)

## 4. 🗄️ Supabase 준비 (1회)

1. [supabase.com](https://supabase.com) 프로젝트 생성 (Region: **Northeast Asia / Seoul** 권장)
2. **SQL Editor** 에 [`supabase/schema.sql`](supabase/schema.sql) **전체**를 붙여넣고 **Run**
   - 생성: `visits`, `surveys`, `staff_events` 테이블 + `people_counts` 함수 + `staff-photos` 스토리지 버킷
3. **Settings → API** 에서 `Project URL` + `service_role`(secret) 키 → 환경변수에 입력

이미 운영 중인 DB가 있다면 `supabase/` 폴더의 `migration_*.sql` 로 증분 적용 가능.

### 알아둘 점
- 모든 테이블 **RLS 켜짐 + 정책 없음** → 익명/anon 키로는 접근 불가, **service_role(서버)** 로만 읽고 씀.
- 직원 사진은 `staff-photos` 버킷(공개 읽기)에 저장, URL 만 DB에 보관.
- Supabase 무료 플랜은 **7일 미사용 시 자동 일시정지** → 운영 시작(7/1) 직전에 생성 권장. (운영 중엔 매일 접속되니 정지 안 됨)
- REST 집계함수가 비활성이라 인원 합계는 `people_counts` **DB 함수(RPC)** 로 계산 (schema.sql에 포함).

## 5. 🚀 배포 (www.ilsanbeachgym.com 과 함께)

이 앱은 **별도 Next.js 서버**가 필요합니다(정적 사이트 아님). 메인 사이트와 같은 도메인의 **서브도메인** 으로 붙이는 걸 권장합니다.

**권장 구성 (예시)**
- 메인 사이트: `www.ilsanbeachgym.com` (기존 그대로)
- 이 앱: `kiosk.ilsanbeachgym.com` → 키오스크 `/`, 관리자 `/admin`, 설문 `/survey`, 직원 `/staff`

**Vercel 기준 절차** (무료/Hobby 또는 Pro)
1. 이 프로젝트를 GitHub 저장소에 푸시
2. Vercel → New Project → 저장소 Import (프레임워크 자동 인식: Next.js)
3. **Environment Variables** 에 3번 항목 입력 후 Deploy
4. **Settings → Domains** 에서 `kiosk.ilsanbeachgym.com` 추가
5. 도메인 DNS 관리처(가비아/Cloudflare 등)에서 Vercel이 안내하는 **CNAME 레코드** 추가
6. 배포 후 `NEXT_PUBLIC_SURVEY_URL` 등 도메인 의존 값을 실제 주소로 맞춰 재배포

> Cloudflare Pages / Render 등 Next.js(SSR/Route Handler) 지원 호스팅이면 어디든 가능. 빌드: `npm run build`, 출력: Next.js (standalone), Node 20+.
> 메인 사이트와 **다른 플랫폼**이어도 서브도메인이면 충돌 없음. 같은 도메인 하위경로(`/kiosk`)로 묶으려면 리버스 프록시가 필요해 비권장.

## 6. 📱 태블릿 키오스크 전체화면 (브라우저 UI 숨김)

- **간단**: 크롬에서 주소 열고 **설치(PWA)** → 탭·주소창 없는 전체화면 앱으로 실행 (매니페스트 `display: fullscreen` 적용됨). 시작 화면 터치 시 자동 전체화면.
- **확실**: 크롬 바로가기 속성 → 대상 뒤에 `--kiosk "URL"` 추가 후 실행. 종료 `Alt+F4`.
- **무인 잠금**: 윈도우 **키오스크(할당된 액세스)** 또는 안드로이드 키오스크 런처.
- 태블릿: 화면 자동꺼짐 OFF, 가로 고정 권장.

## 7. ✅ 현재 상태 / 남은 작업

**완료**
- 키오스크 5단계(언어→안전동의·서명→정보입력→이용안내→완료, 카톡 발송 연동 레이어)
- 관리자: 오늘/전체 통계(인원=동반인원 합계), 성별·연령대·지역·내외국인 분포, 설문 집계, 직원 기록
- 설문(시설/클래스 분기), 직원(출근·업무·현황사진·퇴근, Supabase Storage 사진)
- PWA 전체화면, KST 기준 집계
- **키오스크 모바일 반응형 최적화**(2026-07) — 원래 가로형 태블릿 전용 레이아웃이었으나
  모바일 세로 화면에서도 정상 동작하도록 mobile-first로 재구성. 랜드스케이프(키오스크
  본체) 렌더는 `md:` 브레이크포인트로 그대로 보존.
- **정식 해파랑 로고 적용**(2026-07) — 손코딩 근사 심벌 대신 `public/brand/`의 정식
  로고 자산(심벌·가로/세로 워드마크)을 키오스크 대기화면·헤더에 적용. 헤더는 모든
  화면 폭에서 가로 로고(심벌+워드마크)로 통일. 챌린지·설문·스태프·관리자는 기존
  `src/components/Logo.tsx`(손코딩 심벌) 계속 사용.
- 챌린지 화면 `point` 색 토큰 미정의 버그 수정("실패" 버튼·컷오프 경고 색 정상화)
- **안드로이드 모바일 스크롤 버그 수정**(2026-07) — 안전동의 등 화면에서 아이폰은
  스크롤되는데 안드로이드/카카오톡 인앱브라우저는 안 되던 문제. `.kiosk-root`가
  `min-h-dvh`(최소 높이만 지정)라 콘텐츠에 맞춰 박스 자체가 계속 자라나 스크롤할
  대상이 없어지던 게 원인 — `h-dvh`(고정 높이) + `overflow-y-auto`로 통일해 해결.

**운영 측이 채워야 할 값**
1. **Supabase** 프로젝트 생성 + schema.sql 실행 + 키 입력
2. **관리자 비번**(`ADMIN_PASSWORD`) / `ADMIN_SECRET`
3. **카카오 알림톡**: 솔라피 API키 + 발신번호 + 비즈채널 pfId + 승인 템플릿 ID
   - 등록할 템플릿 예시는 [README.md](README.md) 4번 참고 (변수명이 코드와 일치해야 함: `#{이름}` `#{홈페이지}` `#{예약}` `#{설문}` `#{가이드}`)
4. **외부 링크** 4종 (`NEXT_PUBLIC_*_URL`) — 특히 설문 링크는 배포 도메인 기준

**코드에서 쉽게 바꾸는 값** ([`src/lib/config.ts`](src/lib/config.ts))
- 운영 기간/시간(매일 10:00–20:00)·정원, 직원 체크리스트 카테고리(`STAFF_TASK_CATEGORIES` 등)
- 이용 안내·동의서 문구: [`src/lib/i18n.ts`](src/lib/i18n.ts) / 컬러·폰트: [`src/app/globals.css`](src/app/globals.css)
